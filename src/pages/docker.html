Introducción a Docker

Docker es una plataforma abierta para desarrollar, enviar y ejecutar aplicaciones en contenedores. Los contenedores permiten empaquetar una aplicación con todas sus dependencias en una unidad estandarizada para el desarrollo.

¿Qué es un contenedor?

Un contenedor es una unidad de software ligera y ejecutable que incluye todo lo necesario para ejecutar una aplicación: código, runtime, herramientas del sistema, bibliotecas y configuraciones.

Beneficios de Docker

Consistencia

Funciona igual en desarrollo, pruebas y producción

Aislamiento

Aplicaciones independientes con sus propias dependencias

Eficiencia

Menos recursos que máquinas virtuales, inicio más rápido

Portabilidad

Ejecutable en cualquier sistema con Docker instalado

Docker vs Máquinas Virtuales

Característica Docker Máquina Virtual
Tamaño MB (solo app y dependencias) GB (SO completo)
Rendimiento Casi nativo Overhead por hipervisor
Inicio Segundos Minutos
Aislamiento Nivel de proceso Nivel de hardware

Nota: Docker no es un reemplazo para VMs en todos los casos. Ambos tienen sus usos específicos y pueden complementarse.

Instalación y Configuración

Docker está disponible para los principales sistemas operativos. Aquí te mostramos cómo instalarlo:

Instalar Docker en Linux (Ubuntu/Debian)

$ sudo apt-get update
$ sudo apt-get install \
  apt-transport-https \
  ca-certificates \
  curl \
  gnupg \
  lsb-release

$ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

$ echo \
  "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

$ sudo apt-get update
$ sudo apt-get install docker-ce docker-ce-cli containerd.io

# Verificar instalación
$ sudo docker run hello-world

Instalar Docker en Windows/macOS

Para Windows y macOS, se recomienda descargar Docker Desktop:

  1. Visita docker.com/products/docker-desktop
  2. Descarga el instalador para tu sistema operativo
  3. Sigue las instrucciones del instalador
  4. Verifica la instalación abriendo una terminal y ejecutando:
$ docker --version
$ docker run hello-world

Configuración post-instalación

En Linux, para evitar usar sudo con Docker:

$ sudo groupadd docker
$ sudo usermod -aG docker $USER
$ newgrp docker
# Verificar que funciona sin sudo
$ docker run hello-world

Seguridad: El grupo docker equivale a root. Solo añade usuarios de confianza a este grupo.

Conceptos Fundamentales

Para trabajar efectivamente con Docker, es importante entender sus componentes clave:

Imágenes

Plantillas de solo lectura con las instrucciones para crear un contenedor. Se componen de capas.

Contenedores

Instancias ejecutables de imágenes. Son efímeros por naturaleza.

Volúmenes

Mecanismo para persistir datos generados por contenedores.

Redes

Permiten la comunicación entre contenedores y con el exterior.

Ciclo de vida de un contenedor

Estados posibles de un contenedor Docker

Registro Docker

Repositorio para almacenar y distribuir imágenes. Docker Hub es el registro público predeterminado.

  • Imágenes oficiales: Proporcionadas por proveedores (nginx, postgres, node)
  • Imágenes públicas: Compartidas por la comunidad
  • Imágenes privadas: Para uso interno de organizaciones

Consejo: Siempre verifica las imágenes de terceros antes de usarlas en producción. Las imágenes oficiales son las más seguras.

Comandos Esenciales de Docker

La CLI de Docker es la principal herramienta para interactuar con el motor Docker.

Gestión de imágenes

# Descargar una imagen
$ docker pull nginx:latest

# Listar imágenes locales
$ docker images
$ docker image ls

# Eliminar una imagen
$ docker rmi nginx
$ docker image rm nginx

# Buscar imágenes en Docker Hub
$ docker search mysql

Gestión de contenedores

# Ejecutar un contenedor
$ docker run -d -p 8080:80 --name my-nginx nginx

# Listar contenedores en ejecución
$ docker ps
# Listar todos los contenedores (incluyendo detenidos)
$ docker ps -a

# Detener un contenedor
$ docker stop my-nginx
# Iniciar un contenedor detenido
$ docker start my-nginx

# Eliminar un contenedor
$ docker rm my-nginx
# Eliminar contenedores detenidos
$ docker container prune

# Ver logs de un contenedor
$ docker logs my-nginx
# Seguir logs en tiempo real
$ docker logs -f my-nginx

Opciones comunes de docker run

Opción Descripción Ejemplo
-d Ejecutar en segundo plano (detached) docker run -d nginx
-p Mapear puertos (host:contenedor) docker run -p 8080:80 nginx
-v Montar volúmenes (host:contenedor) docker run -v /datos:/app/data nginx
-e Establecer variables de entorno docker run -e "VAR=valor" nginx
--name Asignar nombre al contenedor docker run --name mi-nginx nginx
--rm Eliminar contenedor al detenerse docker run --rm nginx

Atajo: Puedes combinar múltiples opciones en un solo comando, por ejemplo: docker run -d -p 8080:80 --name my-nginx -v ./html:/usr/share/nginx/html nginx

Crear Imágenes con Dockerfile

Un Dockerfile es un archivo de texto con instrucciones para construir una imagen Docker personalizada.

Ejemplo básico de Dockerfile para Node.js

FROM node:14-alpine
# Establecer directorio de trabajo
WORKDIR /app

# Copiar archivos de package
COPY package*.json ./

# Instalar dependencias
RUN npm install

# Copiar el resto de archivos
COPY . .

# Puerto expuesto
EXPOSE 3000

# Comando para ejecutar la app
CMD ["npm", "start"]

Instrucciones comunes del Dockerfile

Instrucción Propósito Ejemplo
FROM Imagen base a utilizar FROM ubuntu:20.04
RUN Ejecutar comandos durante la construcción RUN apt-get update
COPY Copiar archivos del host a la imagen COPY . /app
ADD Similar a COPY pero con funcionalidades adicionales ADD https://example.com/file.tar.gz /tmp
WORKDIR Establecer directorio de trabajo WORKDIR /app
EXPOSE Documentar puertos expuestos EXPOSE 80
ENV Establecer variables de entorno ENV NODE_ENV=production
CMD Comando por defecto al ejecutar el contenedor CMD ["npm", "start"]
ENTRYPOINT Configura el ejecutable del contenedor ENTRYPOINT ["python"]

Construir y ejecutar una imagen personalizada

# Construir la imagen (desde el directorio con Dockerfile)
$ docker build -t my-node-app .

# Ejecutar la imagen construida
$ docker run -d -p 3000:3000 my-node-app

# Ver imágenes disponibles
$ docker images

# Subir imagen a Docker Hub
$ docker tag my-node-app usuario/my-node-app
$ docker push usuario/my-node-app

Optimización: Ordena las instrucciones en el Dockerfile desde las menos cambiantes a las más cambiantes para aprovechar el cache de Docker.

Orquestación con Docker Compose

Docker Compose permite definir y ejecutar aplicaciones multi-contenedor con un solo archivo YAML.

Ejemplo de docker-compose.yml para app web + base de datos

version: '3.8'

services:
  web:
    build: .
    ports:
      - "5000:5000"
    volumes:
      - .:/code
    environment:
      FLASK_ENV: "development"
    depends_on:
      - redis

  redis:
    image: "redis:alpine"
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data

volumes:
  redis_data:

Comandos esenciales de Docker Compose

# Construir y ejecutar los servicios
$ docker-compose up
# Ejecutar en segundo plano
$ docker-compose up -d

# Detener y eliminar contenedores
$ docker-compose down

# Listar servicios
$ docker-compose ps

# Ver logs
$ docker-compose logs
# Ver logs de un servicio específico
$ docker-compose logs web

# Construir imágenes sin ejecutar
$ docker-compose build

# Ejecutar un comando en un servicio
$ docker-compose exec web bash

Ventajas de Docker Compose

  • Definición centralizada: Todos los servicios en un solo archivo
  • Variables de entorno: Soporte para archivos .env
  • Redes aisladas: Comunicación segura entre contenedores
  • Volúmenes compartidos: Fácil configuración de persistencia
  • Escalado: Posibilidad de escalar servicios fácilmente

Nota: Para entornos de producción con múltiples hosts, considera usar Docker Swarm o Kubernetes en lugar de solo Docker Compose.

Redes en Docker

Docker proporciona varios tipos de redes para permitir la comunicación entre contenedores y con el exterior.

Tipos de redes en Docker

Bridge

Red privada interna por defecto. Los contenedores pueden comunicarse entre sí.

Host

Elimina el aislamiento de red, el contenedor usa la red del host.

Overlay

Permite la comunicación entre contenedores en diferentes hosts (requiere Swarm).

Macvlan

Asigna una dirección MAC a un contenedor, permitiendo que aparezca como un dispositivo de red independiente.
